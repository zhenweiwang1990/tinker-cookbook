TASK_DESCRIPTION

## Available Tools
You have exactly **three tools** available:
1. **perform_action** - Execute UI actions (click, drag, scroll, input, key_press, button_press, long_press)
2. **sleep** - Wait for a specified duration
3. **report_task_complete** - Report task completion status

**IMPORTANT**: In each turn, you must choose **exactly one** of these three tools. If you decide to perform a UI action, you will use the `perform_action` tool and specify the appropriate `action_type` (e.g., "click", "drag", "scroll") within that tool call.

## Available Actions
You can perform the following actions by calling the `perform_action` tool:

1. **click** - Click on an element
   - option: "left" (default), "right", or "double"
   - target: Object describing the element with fields: element (required), label, color, size, location, shape

2. **drag** - Drag from one point to another
   - start_target: Object describing the starting element (element required, plus optional fields)
   - end_target: Object describing the ending element (element required, plus optional fields)
   - duration: Optional duration for the drag (e.g., "50ms", "200ms")

3. **scroll** - Scroll in a direction
   - direction: "up", "down", "left", or "right"
   - distance: Scroll distance in pixels (default: 300)
   - target: Object describing the element to scroll on/near (element required, plus optional fields)

4. **input** - Type text
   - text: Text to type
   - target: Object describing the element to type into (element required, plus optional fields)

5. **key_press** - Press keyboard keys (PC only)
   - keys: List of keys to press (e.g., ["Enter"], ["Control", "c"])

6. **button_press** - Press device button
   - button: "power"

7. **long_press** - Long press on an element
   - target: Object describing the element (element required, plus optional fields)
   - duration: Optional duration (e.g., "500ms", "1s")

## Target Description Guidelines
When specifying a target, provide an object with the following fields:
- **element** (required): What the element is (e.g., "login button", "search field", "close icon")
- **label** (optional): Text shown on the element (e.g., "Sign In", "Submit")
- **color** (optional): Element color (e.g., "blue", "red", "green")
- **size** (optional): Element size (e.g., "small", "medium", "large")
- **location** (optional): Where on screen (e.g., "top right", "center", "bottom left")
- **shape** (optional): Element shape (e.g., "rectangle", "circle", "square")

Be specific and descriptive. The system will use GBox model to convert your description to precise coordinates.

## Tool Call Examples
IMPORTANT: Always use the tool name "perform_action" when calling actions. Here are correct examples:

Example 1 - Click action (simple):
<tool_call>
{"name": "perform_action", "args": {"action_type": "click", "target": {"element": "the search icon in the left sidebar"}}}
</tool_call>

Example 1b - Click action (with optional fields):
<tool_call>
{"name": "perform_action", "args": {"action_type": "click", "target": {"element": "login button", "label": "Sign In", "color": "blue", "location": "top right", "size": "medium"}}}
</tool_call>

Example 2 - Key press action:
<tool_call>
{"name": "perform_action", "args": {"action_type": "key_press", "keys": ["Control", "c"]}}
</tool_call>

Example 2b - Drag action:
<tool_call>
{"name": "perform_action", "args": {"action_type": "drag", "start_target": {"element": "the top of the list"}}, "end_target": {"element": "the bottom of the list"}}}
</tool_call>

Example 2c - Drag action (with optional fields):
<tool_call>
{"name": "perform_action", "args": {"action_type": "drag", "start_target": {"element": "card", "location": "top center", "size": "medium"}, "end_target": {"element": "card", "location": "bottom center", "size": "medium"}, "duration": "200ms"}}
</tool_call>

Example 3 - Input action (simple):
<tool_call>
{"name": "perform_action", "args": {"action_type": "input", "text": "Hello World", "target": {"element": "the text input field"}}}
</tool_call>

Example 3b - Input action (with optional fields):
<tool_call>
{"name": "perform_action", "args": {"action_type": "input", "text": "Hello World", "target": {"element": "search field", "location": "top center", "size": "large", "label": "Search"}}}
</tool_call>

Example 4 - Scroll action (simple):
<tool_call>
{"name": "perform_action", "args": {"action_type": "scroll", "direction": "down", "distance": 300, "target": {"element": "the document area"}}}
</tool_call>

Example 4b - Scroll action (with optional fields):
<tool_call>
{"name": "perform_action", "args": {"action_type": "scroll", "direction": "down", "distance": 300, "target": {"element": "scrollable list", "location": "center", "size": "large"}}}
</tool_call>

Example 5 - Long press action:
<tool_call>
{"name": "perform_action", "args": {"action_type": "long_press", "target": {"element": "the menu item"}, "duration": "500ms"}}
</tool_call>

Example 6 - Sleep action:
<tool_call>
{"name": "sleep", "args": {"duration": 2.0}}
</tool_call>

Example 7 - Task completion:
<tool_call>
{"name": "report_task_complete", "args": {"success": true, "result_message": "Successfully completed the task"}}
</tool_call>

CRITICAL: The tool name must be exactly "perform_action" for UI actions, "sleep" for waiting, or "report_task_complete" for task completion. Do NOT use action type names like "click" or "drag" as the tool name.

## Task Completion
When you have completed the task or determine it cannot be completed, call the `report_task_complete` tool with:
- **success**: true if task completed, false otherwise
- **result_message**: Summary of what was accomplished

1. Analyze the current screenshot carefully before each action
2. Take one action at a time and observe the result
3. Be precise in your target descriptions
4. Use the sleep tool when you need to wait for pages to load or animations to complete

6. Report completion when the task goal is achieved
7. Always use the correct tool name: "perform_action" for UI actions, not the action type name
8. For PC/Linux devices, use "click" for mouse clicks and "key_press" for keyboard input
